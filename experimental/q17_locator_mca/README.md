# q=17 Locator-Fiber and MCA Exhaustive Check

This directory gives a reproducible finite check for the good-first item:

> Exhaust `q=17`, `n=16`, `rho=1/2,1/4` for locator fibers and MCA bad
> slopes.

The domain is `D = F_17^*`, generated by `3`. For each rate and every quotient
scale `N` with `a = n/N` dividing `k`, the checker reports:

- the locator restricted-sum fibers for slack `t=1` and `t=2`;
- the exact slopes for which the canonical received word is close to the
  relevant locator-list code;
- the exact support-wise MCA-bad slopes for the same canonical line against
  `RS[F_17,D,k]`.

The split between slack values matters. Slack one is a direct MCA witness for
`RS[F,D,k]`. Slack two is the Paper D list-fiber input for `C+ = RS[F,D,k+1]`;
the checker still exhausts direct MCA at slack two, but does not assume the
slack-two locator fibers are MCA witnesses for `C`.

## Reproduction

```sh
python3 experimental/q17_locator_mca/verify_q17_locator_mca.py \
  --check experimental/q17_locator_mca/q17_locator_mca_certificate.json
```

To regenerate the certificate:

```sh
python3 experimental/q17_locator_mca/verify_q17_locator_mca.py \
  --output experimental/q17_locator_mca/q17_locator_mca_certificate.json
```

The verifier uses only the Python standard library. It does not enumerate all
codewords. For each support it builds finite-field parity checks for the
degree-`<d` evaluation space and then exhausts all supports and all `17` slopes.
