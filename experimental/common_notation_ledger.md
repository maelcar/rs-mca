## Purpose

Milestone M0 asks for a shared notation file covering the symbols and field
conventions used across the four manuscripts. This ledger is a review artifact:
it records the proposed common meanings, notes known drift, and lists promotion
checks before any notation edits are made in `tex/`.

## Default Conventions

- `F`, `B`: ambient field `F` and generated/base subfield `B`; write
  `D subset B` when a domain is base-rational.
- `q`: size of the ambient field currently used by the code or theorem.
- `q_gen`: size of the field generated by the evaluation domain and locator
  coefficients.
- `q_line`: field from which affine-line, CA, MCA, or line-decoding slopes are
  sampled.
- `q_chal`: verifier challenge field used by the protocol.
- `D`: evaluation domain. Default target convention: multiplicative coset,
  with subgroup as the special case.
- `n`: domain length, `n = |D|`.
- `k`: RS dimension, with polynomials of degree `< k`.
- `rho`: rate, `rho = k/n`.
- `delta`: proximity radius. Always state whether it is field/list, internal,
  or support-wise.
- `eta`: capacity reserve, `eta = 1 - rho - delta`.
- `N`: quotient order.
- `a`: fiber size in a quotient map, usually `a = n/N`.
- `sigma`: slack or reserve in symbols, depending on paper context; define
  locally if both meanings appear.
- `mu`: protocol or list interleaving arity.
- `nu`: implementation interleaving or repetition factor consumed by MCA/list
  transfer.
- `Lhat_mu(delta)`: certified upper bound for a `mu`-interleaved list size at
  radius `delta`.
- `CA`: correlated agreement, a line-slope condition with a proximity-loss
  convention.
- `MCA`: mutual correlated agreement, usually support-wise unless otherwise
  stated.
- Support-wise bad slope: a slope with a large explaining support for
  `f + z g` but no common support explanation for `f,g`.
- Line-decoding: stronger collinearity or decoding-assignment statement; do
  not collapse it into MCA without a theorem.
- Extension code: code over `F` evaluated on `D subset B`; list accounting may
  reduce to interleaved base-code accounting.
- Proof status: one of `PROVED`, `CONDITIONAL`, `CONJECTURAL`,
  `EXPERIMENTAL`, `AUDIT`, or `COUNTEREXAMPLE`.

## Field Ledger Rules

1. Use `q_gen` for generated-field entropy and locator-fiber ledgers unless the
   code and locator data are genuinely lifted.
2. Use `q_line` for list-over-field and MCA/line denominators only when the
   theorem or protocol reduction samples slopes over that field.
3. Use `q_chal` only for verifier challenge sampling. Do not use it to pay a
   `q_gen` entropy bill.
4. If `q_line > q_gen`, record both the extension-code list identity and the
   extension-line MCA status.
5. For `D subset B subset F`, base-rational lines have confined bad slopes;
   genuinely `F`-valued line data is a separate F1 issue.

## Radius and Slack Rules

| Quantity | Hygiene rule |
| --- | --- |
| `delta` | Print the exact interval and endpoint convention. |
| `delta_fld`, `delta_int` | Use two symbols when CA has proximity loss. |
| `eta` | Define by `eta = 1-rho-delta`; positive `eta` means below capacity. |
| `t` | If used as query count and as locator slack in the same note, rename one occurrence. |
| `sigma` | If used for both symbol reserve and quotient slack, add a local glossary row. |
| `a` | For quotient maps, reserve `a` for `n/N` unless a paper section explicitly overrides it. |

## Domain Convention

Default target convention:

```text
D is a multiplicative coset in a generated field B, with subgroup domains
handled by taking the coset representative to be 1.
```

Known drift to resolve before promotion:

- Paper A and Paper B often state results for subgroups.
- Paper D allows multiplicative cosets, matching the survey convention.
- A promoted common notation file should state cosets by default and identify
  exactly which lemmas still require the subgroup special case.

## Object-Separation Checklist

Do not merge the following objects unless a cited theorem identifies them:

| Object A | Object B | Required bridge |
| --- | --- | --- |
| Base-code list size | Interleaved-list size | Interleaving theorem or enumeration. |
| Extension-code list size | Base-code list size | Extension-code/list identity. |
| CA | MCA | Directional implication with matching radii. |
| MCA | Line-decoding | Line-decoding theorem or reduction. |
| Affine-line MCA | Curve or polynomial-generator MCA | Curve-family theorem. |
| `q_gen` entropy | `q_line` or `q_chal` denominator | Genuine lift theorem. |

## Proof-Status Hygiene

Every ledger, certificate, or theorem-reference table should label each row:

| Status | Meaning |
| --- | --- |
| `PROVED` | The statement is proved in the manuscripts or by a checked finite certificate. |
| `CONDITIONAL` | The result depends on a named imported theorem or assumption. |
| `CONJECTURAL` | The statement is an explicit conjecture or unproved model. |
| `EXPERIMENTAL` | The item is a heuristic scan, prototype, or unpromoted computation. |
| `AUDIT` | The item is a checklist or review note, not a proof authority. |
| `COUNTEREXAMPLE` | The item records a verified obstruction or disproof instance. |
