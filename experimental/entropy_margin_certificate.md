# Generated-Field Entropy Margin Certificate

**Status:** `PROVED` for the arithmetic interval certificate below.
Scripted parameter runs are `AUDIT`, not theorem upgrades.

**Target:** Generated-field entropy ledger for Paper C reserve certificates.

**Companion script:** `experimental/entropy_margin.py`

## Claim

For finite parameters `n`, `k`, `sigma`, and `q_gen`, set

```text
a = k + sigma.
```

The generated-field entropy ledger is

```text
M_ent = sigma * log2(q_gen) - log2(binomial(n, a)).
```

This is the finite arithmetic quantity appearing in the generated-field
reserve check. It uses `q_gen`, the field generated by the domain and locator
data. It must not be paid with `q_line` or `q_chal` unless a separate theorem
identifies the relevant objects.

## Status

`PROVED` for the following interval certificate:

if `p = a/n` and `0 < a < n`, then

```text
n H2(p) - log2(n+1)
  <= log2(binomial(n, a))
  <= n H2(p),
```

where

```text
H2(p) = -p log2(p) - (1-p) log2(1-p).
```

Consequently, with

```text
L = n H2(p) - log2(n+1),
U = n H2(p),
A = sigma * log2(q_gen),
```

we have

```text
A - U <= M_ent <= A - L.
```

The endpoint cases `a=0` and `a=n` have `log2(binomial(n,a)) = 0`.

## Parameters

Required script inputs:

```text
n          domain size
k          Reed--Solomon dimension
sigma      reserve in agreement symbols
q_gen      generated field size, or log2(q_gen)
```

Optional inputs:

```text
Gamma_ent  additive entropy margin target, in bits
B          polynomial list exponent for a target n^B
```

For a polynomial target `n^B`, the script also reports

```text
M_B = M_ent - B log2(n).
```

## Proof

For `0 < a < n`, let `p=a/n`. The standard counting proof for the binary
entropy upper bound gives

```text
binomial(n, a) <= 2^(n H2(p)).
```

For the lower bound, evaluate the Bernoulli distribution with success
probability `p=a/n`. The probability of exactly `a` successes is

```text
binomial(n, a) * p^a * (1-p)^(n-a)
  = binomial(n, a) * 2^(-n H2(p)).
```

There are only `n+1` possible Hamming weights, and the weight `a` is a maximum
probability weight for this Bernoulli distribution. Hence this probability is
at least `1/(n+1)`, so

```text
binomial(n, a) >= 2^(n H2(p)) / (n+1).
```

Taking base-2 logarithms gives the interval for
`log2(binomial(n,a))`. Subtracting this interval from
`A = sigma * log2(q_gen)` reverses the endpoints in the usual way and gives

```text
A - U <= M_ent <= A - L.
```

Therefore:

```text
if A - U >= Gamma_ent, then M_ent >= Gamma_ent;
if A - L <  Gamma_ent, then M_ent <  Gamma_ent;
otherwise the entropy interval alone is inconclusive.
```

The same implication applies to the polynomial-list margin after subtracting
`B log2(n)` from both endpoints.

## Ledger Impact

This check mechanizes one reserve ledger: generated-field entropy. It is a
front-end audit for the later locator/list and MCA ledgers. In particular, it
is useful for catching the common double-crediting error where an extension
challenge field is accidentally used in place of `q_gen`.

## Reproducibility

Example:

```bash
python3 experimental/entropy_margin.py --n 16 --k 8 --sigma 1 --q-gen 17
```

JSON output:

```bash
python3 experimental/entropy_margin.py --n 16 --k 8 --sigma 1 --q-gen 17 --format json
```

KoalaBear-class bit-width audit:

```bash
python3 experimental/entropy_margin.py --n 2097152 --k 1048576 --sigma 131072 --q-gen-log2 31
```

Every run is an `AUDIT` result. Passing this certificate means only that the
generated-field entropy margin clears the requested arithmetic check.
